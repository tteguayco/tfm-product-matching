1.1.1 Random forest simple (1 punto)¶

La idea bsica del bagging es utilizar el conjunto de entrenamiento original para generar centenares o miles de conjuntos similares usando muestreo con reemplazo. En este concepto est basado el algoritmo Random Forest, la combinacin de varios rboles de decisin, cada uno entrenado con una realizacin diferente de los datos. La decisin final del clasificador combinado (la Random Forest) se toma por mayora, dando el mismo peso a todas las decisiones parciales tomadas por los clasificadores base (los rboles).

1.1.2 Out-of-bag (1 punto)¶

Una ventaja del bagging usado en el Random Forest es que cada uno de los rboles de decisin ha sido entrenado con una combinacin diferente de los datos (muestreo con reemplazo), o sea que cada uno de los rboles no ha visto una determinada parte de los datos originales. Esto define una especie de conjunto de test para cada uno de los rboles, llamado out-of-bag, que puede ser usado para estimar el error del modelo sin necesidad de usar el conjunto de test real que creamos previamente, ni de usar estrategias de cross-validation.

Implementacin: Usando los conjuntos *X_train_pca* e *y_train_pca*, entrenar un modelo Random Forest con 100 rboles de decisin. Mostrar la precisin de este modelo en el *out-of-bag* y en el conjunto *X_test_pca*.
Sugerencia: usar el mdulo *RandomForestClassifier* de sklearn. Para aprender ms sobre *out-of-bag* y sobre como usar este mdulo (incluyendo el atributo *oob_score_*), os recomendamos los siguientes enlaces:
http://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestClassifier.html
http://scikit-learn.org/stable/auto_examples/ensemble/plot_ensemble_oob.html

1.1.3 Probabilidad por clase (1 punto)¶

Otra ventaja del bagging usado en el Random Forest es que cada uno de los rboles de decisin, entrenado con una combinacin diferente de los datos, puede obtener un resultado diferente. En los problemas de clasificacin, el resultado de cada rbol se considera como un voto diferente, y la prediccin final del modelo es la clase que haya obtenido ms votos teniendo en cuenta todos los rboles.

Estos votos individuales de los rboles tambin se pueden usar para estimar la probabilidad con la que el modelo prev cada una de las clases, siendo la probabilidad para cada clase igual al nmero de votos obtenidos para aquella clase dividido entre el nmero de rboles.

from sklearn.model_selection import learning_curve

def plot_learning_curve(estimator, title, X, y, ylim=None, cv=None,
                        n_jobs=1, train_sizes=np.linspace(.1, 1.0, 5)):
    """
    Generate a simple plot of the test and training learning curve.

    Parameters
    ----------
    estimator : object type that implements the "fit" and "predict" methods
        An object of that type which is cloned for each validation.

    title : string
        Title for the chart.

    X : array-like, shape (n_samples, n_features)
        Training vector, where n_samples is the number of samples and
        n_features is the number of features.

    y : array-like, shape (n_samples) or (n_samples, n_features), optional
        Target relative to X for classification or regression;
        None for unsupervised learning.

    ylim : tuple, shape (ymin, ymax), optional
        Defines minimum and maximum yvalues plotted.

    cv : int, cross-validation generator or an iterable, optional
        Determines the cross-validation splitting strategy.
        Possible inputs for cv are:
          - None, to use the default 3-fold cross-validation,
          - integer, to specify the number of folds.
          - An object to be used as a cross-validation generator.
          - An iterable yielding train/test splits.

        For integer/None inputs, if ``y`` is binary or multiclass,
        :class:`StratifiedKFold` used. If the estimator is not a classifier
        or if ``y`` is neither binary nor multiclass, :class:`KFold` is used.

        Refer :ref:`User Guide <cross_validation>` for the various
        cross-validators that can be used here.

    n_jobs : integer, optional
        Number of jobs to run in parallel (default 1).
    """
    plt.figure()
    plt.title(title)
    #if ylim is not None:
    #    plt.ylim(*ylim)
    plt.xlabel("Training examples")
    plt.ylabel("Score")
    train_sizes, train_scores, test_scores = learning_curve(
        estimator, X, y, cv=cv, n_jobs=n_jobs, train_sizes=train_sizes)
    train_scores_mean = np.mean(train_scores, axis=1)
    train_scores_std = np.std(train_scores, axis=1)
    test_scores_mean = np.mean(test_scores, axis=1)
    test_scores_std = np.std(test_scores, axis=1)
    plt.grid()

    plt.fill_between(train_sizes, train_scores_mean - train_scores_std,
                     train_scores_mean + train_scores_std, alpha=0.1,
                     color="r")
    plt.fill_between(train_sizes, test_scores_mean - test_scores_std,
                     test_scores_mean + test_scores_std, alpha=0.1, color="g")
    plt.plot(train_sizes, train_scores_mean, 'o-', color="r",
             label="Training score")
    plt.plot(train_sizes, test_scores_mean, 'o-', color="g",
             label="Test score")

    plt.legend(loc="best")
    return plt
